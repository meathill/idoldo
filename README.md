Idoldo
===

我想开发一个游戏平台，名字叫做“爱豆豆”。在这个平台上，有很多做好的小游戏，用户可以上传自己喜欢的“爱豆”，组装成应援小游戏。比如，一个连连看游戏，用户可以上传某明星的 20 张图片，然后就能生成一个某明星的连连看小游戏。接着，这个用户可以把这个小游戏分享到社交媒体，然后其他粉丝就可以玩到这个连连看小游戏。相当于我们给用户一个低成本的跟同好互动的平台。

其他用户不需要登录就能玩到小游戏，但需要登录才能创建自己的游戏。想要记录游戏历史也需要创建账号，总之我们要在合适的地方才要求登录。

我们会不断推出新游戏，也会给已有游戏增加更多可选项，让用户可以享受到更丰富的游戏过程。

关于收费我还没想好，目前有一些想法：

1. 加广告
2. 和官方合作
3. 引入体力机制，用户需要消耗体力才能创建游戏和游玩更多游戏

平台方面，我希望先用 iOS，但我没想好怎么解决分享后即开即玩，也许可以用那个什么 clip？


项目目录
---

- packages/website: 网站，目前是个 waitlist landing page
- packages/api: API 服务，预备使用 hono 提供 API
- packages/app: App，使用 React Native + Expo 构建


技术架构
---

- 全部使用 TypeScript，降低开发成本，复用代码
- 全部使用 TailwindCSS
- 服务器使用 Cloudflare Workers + Cloudflare 全家桶
- 应用暂时只支持 iOS
